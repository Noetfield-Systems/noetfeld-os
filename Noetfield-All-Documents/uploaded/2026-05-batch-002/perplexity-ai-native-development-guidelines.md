# AI-Native Development System Guidelines - Perplexity

Document key: `perplexity-ai-native-development-guidelines`

Source: Perplexity

Status in source: engineering guidelines and design principles

## Normalized purpose

This document frames AI tooling as a distributed execution system rather than a
set of editor extensions.

It introduces the concept of a Unified State Model (USM), state
synchronization, multi-agent coordination, policy-driven orchestration,
cognitive latency optimization, trust infrastructure, persistent memory, and
state transition contracts.

## Key principles

- IDEs are execution surfaces.
- Agents are execution units.
- Models are cognition engines.
- The system is a coordinated runtime.
- Context should evolve into executable state.
- Agents must not act on stale or partial state.
- Multi-agent systems fail from coordination gaps, not intelligence gaps.
- Trust infrastructure is required before autonomy can scale.
